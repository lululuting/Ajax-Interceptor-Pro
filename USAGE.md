# 使用说明

## 这是什么

`Ajax Interceptor Pro` 是一个 Chrome 接口拦截插件，可以在前端开发、联调和调试时，直接拦截请求并返回你自定义的响应内容。

它基于 Ajax Interceptor 的思路继续扩展，加入了分组管理、命中计数、导入导出和更完整的可视化界面。

## 安装

1. 执行 `npm install`
2. 执行 `npm run build`
3. 打开 Chrome，进入 `chrome://extensions/`
4. 打开右上角“开发者模式”
5. 选择“加载已解压的扩展程序”
6. 选择项目里的 `dist/` 目录

## 使用流程

### 1. 新建分组

点击左侧“新建分组”，给规则分类。

默认会保留一个 `未分组`，你也可以自己继续加业务分组，比如：
- 用户模块
- 订单模块
- 支付模块

### 2. 添加规则

点击“添加规则”，填写：
- 规则名称
- 请求方法
- URL 匹配模式
- 所属分组
- 返回数据
- 状态码

规则命中后，插件会直接返回你配置的响应内容。

### 3. 查看命中情况

每条规则右侧都有命中计数。命中后会立即更新，方便确认请求是否真的走到了这条规则。

### 4. 导入导出配置

设置里支持导出当前配置，也支持导入已有规则：
- `覆盖`：用导入内容替换当前规则
- `合并`：按分组和规则合并，尽量避免重复

## 两种打开方式

### 弹窗模式

点击浏览器扩展图标，直接打开插件界面。

### DevTools 模式

按 `F12` 打开开发者工具，在 `Ajax拦截` 面板里使用。

## 适合用来做什么

- 本地没有后端时先模拟接口
- 接口字段调试
- 异常返回模拟
- 固定响应复现 bug
- 前端联调前自测

## 注意事项

- URL 匹配支持通配符
- 如果规则没命中，先检查请求方法和 URL 是否一致
- 建议调试时关闭浏览器缓存
- 不用时可以关闭全局开关，避免影响正常页面请求
